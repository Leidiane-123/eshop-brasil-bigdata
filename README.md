Aplicação Prática de Tecnologias de Banco de Dados e Big Data em uma Empresa de Comércio Eletrônico   1. Introdução e Objetivos   Este projeto foi desenvolvido para solucionar os desafios tecnológicos da E-Shop Brasil, uma plataforma que gerencia mais de 5 milhões de clientes ativos e processa 100 mil pedidos diários.  O objetivo central é implementar uma infraestrutura capaz de gerir grandes volumes de dados (Big Data), garantindo escalabilidade, segurança em conformidade com a LGPD e permitindo a personalização da experiência do usuário por meio de análises avançadas de dados.  2. Tecnologias Utilizadas   Docker & Docker Compose: Utilizados para a conteinerização e orquestração dos serviços, garantindo que o ambiente de banco de dados e a aplicação operem de forma isolada e integrada.  MongoDB: Banco de dados NoSQL orientado a documentos, escolhido pela sua alta performance em leituras e escritas massivas e flexibilidade de esquema.  Streamlit: Framework em Python utilizado para construir a interface gráfica que permite a interação humana com o cluster de Big Data.  3. Descrição da Aplicação   A aplicação simula um console de administração para a E-Shop Brasil com as seguintes funcionalidades:Inserção de Dados: Permite a ingestão de novos registros de clientes diretamente no MongoDB.  Manipulação e Edição: Módulo para atualização de valores e exclusão de documentos, garantindo a manutenção do banco.  Consultas e Visualização: Exibição de tabelas e gráficos dinâmicos baseados nos dados armazenados.  Concatenação (Big Data): Implementação de Aggregation Pipelines para cruzar e processar informações de forma complexa.  4. Comandos para Execução   Para rodar este projeto localmente, siga os passos abaixo:Certifique-se de que o Docker e o Docker Compose estão instalados e ativos.No terminal, acesse a pasta raiz do projeto:Bashcd eshop-brasil-bigdata
Execute o comando para construir e iniciar os serviços:Bashdocker-compose up --build
